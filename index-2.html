Introduction

The building blocks before we dive into Events

🐱

What is Scratch?

Scratch is a free, block-based visual programming language created by MIT Media Lab. Instead of typing text-based code, users snap together colorful blocks like puzzle pieces to build games, animations and stories.

💡

Why Scratch?

Scratch removes the fear of syntax errors and lets beginners focus purely on logic and creativity. It is the most popular entry point for children and beginners to learn computational thinking worldwide.

⚡

What are Events?

Events are the golden "Hat Blocks" in Scratch (usually yellow) that sit at the top of a script and wait for something to happen — like a click, a key press, or a message — before running the code below them.

🌍

Where are Events used?

Events power almost everything interactive: starting a game with the Green Flag, jumping when Space is pressed, switching scenes, reacting to sound, and communicating between different sprites using broadcasts.

🏆

Why Events are important?

Without Events, a Scratch project would never start or respond to the user. Events are the bridge between the human user and the program's logic — this same idea powers real-world software, from websites to security systems.

🔐

Events & Cyber Logic

Just like a firewall "listens" for a trigger before acting, Scratch Events listen for a trigger before running code. Understanding Events is your first step toward understanding event-driven systems used in real software and security tools.

Event Blocks

Click. Run. Reset. Learn by doing.

1

When Green Flag Clicked

Explanation: This is the most common Event block. It runs the script the moment the user clicks the green flag button — the universal "Start" button in Scratch.

Real Life Example: Just like pressing the power button turns on a TV, clicking the Green Flag starts the entire Scratch program.

when 🏳 green flag clicked say "Hello World!" for 2 seconds move 10 steps
Summary: Green Flag = the official "Start Program" trigger.
🏳️

Click the flag to run!

2

When This Sprite Clicked

Explanation: This Event triggers only when the specific sprite itself is clicked, allowing individual characters to respond uniquely.

Real Life Example: Like a doorbell button — pressing that particular button rings that particular bell, not the neighbor's.

when this sprite clicked play sound "Meow" change color effect by 25
Summary: Gives each sprite its own independent click-reaction.
🐱

Click the cat sprite!

3

When Space Key Pressed

Explanation: This Event fires whenever a chosen key on the keyboard is pressed, letting the user directly control the sprite.

Real Life Example: Like a car's accelerator pedal — pressing it directly triggers the car to move faster.

when space key pressed change y by 40 play sound "Jump"
Summary: Keyboard Events connect physical input to on-screen action.

4

When Backdrop Switches To

Explanation: This Event runs code when the stage's backdrop changes to a specific chosen backdrop — great for scene-based storytelling.

Real Life Example: Like a theatre stage — when the curtain reveals a new scene, actors change their behaviour accordingly.

when backdrop switches to "Night" say "It's dark now!" switch costume to "sleepy"
Summary: Enables multi-scene projects like stories and levels.
☀ Day Scene
5

When Loudness > Value

Explanation: This Event constantly checks the microphone's loudness, and triggers code once loudness crosses a set threshold.

Real Life Example: Like a "Clap to turn on the lights" device — the device reacts once sound level crosses a limit.

when loudness > 50 change size by 10 play sound "pop"
Summary: Lets projects respond to real-world sound input.
🎤

Simulate a loud sound!

6

Broadcast

Explanation: Broadcast sends a message instantly to all sprites, and immediately continues running the next block without waiting.

Real Life Example: Like shouting an announcement in a hallway and walking away immediately — you don't wait to see who reacts.

broadcast "start game" (script continues immediately)
Summary: Fire-and-forget messaging between sprites.
📡🐢
start game
7

Broadcast and Wait

Explanation: This block sends a message but PAUSES the sender's script until every receiving script has completely finished running.

Real Life Example: Like handing over a baton in a relay race and waiting there until the next runner finishes their lap.

broadcast "load level" and wait say "Level Loaded!"
Summary: Difference from Broadcast — this one waits, Broadcast doesn't.
📡🐢
load level

Sender status: Idle

8

When I Receive

Explanation: This Event is the "listener" hat block — it waits for a specific broadcast message and runs its script only when that message arrives.

Real Life Example: Like a walkie-talkie set to a specific channel — it only reacts to messages sent on that exact channel.

when I receive "start game" show play sound "whistle"
Summary: The receiving half of the Broadcast system.
